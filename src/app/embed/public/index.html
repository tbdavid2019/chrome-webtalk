WebTalk Embed

在任何網站加入 WebTalk

用一段 script 加入與目前頁面訪客互動的 P2P 聊天室。預設建議使用純聊天室版；需要 AI 摘要與頁面對話時，再選擇混合版。

最快接入：純 P2P 聊天

請在每個要開放聊天的頁面輸出不同的 page ID。以下範例適合文章、商品、知識庫與 Share 頁。

<meta name="webtalk-page-id" content="{{此頁穩定且唯一的 ID}}" />
<script
  async
  src="https://你的-vercel-project.vercel.app/webtalk-chat.js"
  data-webtalk-site-id="your-site"
></script>

這會載入 webtalk-chat.js：只有聊天室與必要的暱稱／頭像初始化，不含 AI 按鈕、AI 請求或 extension 設定頁入口。

需要 AI 時：混合版

<meta name="webtalk-page-id" content="{{此頁穩定且唯一的 ID}}" />
<script
  async
  src="https://你的-vercel-project.vercel.app/webtalk.js"
  data-webtalk-site-id="your-site"
  data-webtalk-ai-endpoint="https://你的-vercel-project.vercel.app/api/webtalk/ai"
></script>

webtalk.js 會多出 AI 摘要、頁面對話與聊天室 @ai 功能。使用前請先完成 Vercel AI proxy 的環境變數與來源 allowlist 設定。

手機與版面

桌機為右側可調整寬度的聊天室。手機會變成半螢幕覆蓋層，不修改你的 DOM、CSS 或捲動行為：另一半畫面仍顯示你的網站。 

<script
  src="https://你的-vercel-project.vercel.app/webtalk-chat.js"
  data-webtalk-mobile-placement="bottom"
></script>

bottom（預設）讓網站顯示在上半部、聊天在下半部；設為 top 則聊天在上半部、網站顯示於下半部。

所有參數

參數 可用值 用途
data-webtalk-scope meta、origin、path 房間策略；預設 meta，最安全。
data-webtalk-page-id 穩定頁面 ID 覆寫 webtalk-page-id meta，通常建議使用 meta。
data-webtalk-site-id 穩定字串 網站 namespace；網址遷移後仍可維持房間識別。
data-webtalk-room-id 任意穩定字串 完全指定房間，優先權最高。
data-webtalk-meta-name meta 名稱 改用自訂的 page ID meta 名稱。
data-webtalk-auto-mount true、false 是否載入後立即掛載；可用於登入後才啟用。
data-webtalk-virtual-room true、false 是否加入跨站 presence；預設關閉。
data-webtalk-mobile-placement bottom、top 手機半螢幕覆蓋位置；預設 bottom。
data-webtalk-ai-endpoint URL 只在混合版需要；指定 server-side AI proxy。

房間策略

每個內容頁各自聊天（建議）

使用預設 meta，並讓伺服器針對每頁輸出不同的 webtalk-page-id。

全站共用一個聊天室

<script src="https://你的-vercel-project.vercel.app/webtalk-chat.js"
  data-webtalk-scope="origin"></script>

按網址路徑分房

<script src="https://你的-vercel-project.vercel.app/webtalk-chat.js"
  data-webtalk-scope="path"></script>

/share/a 與 /share/a/ 會視為相同房間。

登入後才掛載

<script src="https://你的-vercel-project.vercel.app/webtalk-chat.js"
  data-webtalk-auto-mount="false"></script>
<script>
  // 在你的登入流程完成後執行
  window.WebTalk.mount({ scope: 'origin', mobilePlacement: 'bottom' })
</script>

隱私與資料流

P2P 聊天：聊天內容透過 WebRTC DataChannel 在使用者瀏覽器間傳輸，訊息與本機歷史不會寫入 Vercel AI proxy。純聊天室版不會發出 AI 請求。
連線建立：WebRTC 仍需要 Artico signaling service 交換 SDP／ICE 連線資訊；它不是聊天室訊息儲存服務。請勿將產品描述為「所有資料完全不經過任何伺服器」。
混合版 AI：當使用者使用摘要、AI 對話或 @ai 時，頁面文字與提問會送往你設定的 Vercel AI proxy，並由該 proxy 呼叫你的 LLM provider。請在隱私政策中揭露此流程。

與 Chrome extension 共存

當使用者已安裝 WebTalk Chrome extension，而你的網站已掛載 WebTalk embed 時，extension 會自行偵測官方 widget 並停止自己的 UI 與連線。站長不需要新增額外判斷，也不需要偵測瀏覽器 extension。

常見問題

沒有出現聊天室？
預設 meta 模式必須有 webtalk-page-id。若你要全站共用，明確改為 data-webtalk-scope="origin"。
網頁有 CSP？
在你的 script-src allowlist 加入 Vercel 網域；混合版也需要在 connect-src 放行 AI endpoint。
AI 不能用？
確認 endpoint、LLM_* 環境變數與 WEBTALK_ALLOWED_ORIGINS 已設定，並使用混合版 webtalk.js。
